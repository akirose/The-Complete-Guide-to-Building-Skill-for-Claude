# 소개

스킬은 클로드에게 특정 작업이나 워크플로를 처리하는 방법을 가르치는 지침 묶음이며, 단순한 폴더 형태로 패키징됩니다. 스킬은 사용자의 필요에 맞게 클로드를 맞춤화하는 가장 강력한 방법 중 하나입니다. 매 대화마다 선호 사항, 절차, 도메인 지식을 다시 설명하는 대신, 스킬을 통해 한 번 가르치고 계속 활용할 수 있습니다.

스킬은 반복 가능한 워크플로가 있을 때 특히 강력합니다. 예를 들어, 명세를 바탕으로 프론트엔드 디자인을 만들거나, 일관된 방법론으로 리서치를 수행하거나, 팀 스타일 가이드를 따르는 문서를 작성하거나, 여러 단계를 가진 프로세스를 오케스트레이션하는 작업에 잘 맞습니다. 스킬은 코드 실행, 문서 생성 같은 클로드의 기본 기능과도 잘 결합됩니다. MCP 통합을 구축하는 경우에는, 스킬이 원시 도구 접근을 신뢰 가능하고 최적화된 워크플로로 전환해 주는 추가 계층이 됩니다.

이 가이드는 효과적인 스킬을 만드는 데 필요한 전 과정을 다룹니다. 계획과 구조 설계부터 테스트와 배포까지 모두 포함합니다. 개인용, 팀용, 커뮤니티 공유용 중 어떤 목적이든 실무적인 패턴과 실제 예시를 찾을 수 있습니다.

## 배우게 될 내용
- 스킬 구조를 위한 기술 요구사항과 모범 사례
- 단독 스킬 및 MCP 강화 워크플로 패턴
- 다양한 사용 사례에서 효과가 좋았던 패턴
- 스킬 테스트, 반복 개선, 배포 방법

## 이 가이드의 대상
- 클로드가 특정 워크플로를 일관되게 따르길 원하는 개발자
- 클로드를 반복 업무에 적극 활용하려는 고급 사용자
- 조직 전반에서 클로드 작업 방식을 표준화하려는 팀

## 이 가이드를 따라가는 두 가지 경로

단독 스킬을 만들고 있다면 `Fundamentals`, `Planning and design`, 그리고 카테고리 1~2에 집중하세요. MCP 통합을 강화하려면 `Skills + MCP` 섹션과 카테고리 3이 핵심입니다. 두 경로는 같은 기술 요구사항을 공유하며, 본인 사용 사례에 맞는 부분을 선택하면 됩니다.

이 가이드를 마치면 한 번의 작업 세션 안에 동작하는 스킬을 만들 수 있게 됩니다. `skill-creator`를 사용하면 첫 스킬의 빌드와 테스트까지 보통 15~30분이면 가능합니다.

이제 시작해봅시다.
